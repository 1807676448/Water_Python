# 神经网络训练与部署说明（BP / PSO-BP，双输出）

## 1. 当前实现目标
本工程当前实现的是双输出水质预测模型：
- 输入：`254`、`550`、`tem`
- 输出：`cod`、`uv254`
- 网络：BP 神经网络（`tansig` + `purelin`），默认 3→3→2 结构（20参数）
- 归一化：`MinMaxScaler(-1, 1)`
- 训练可选：Adam / SGD / LBFGS，PSO 初始化，L2 正则化，早停，学习率调度

## 2. 输出目录统一管理
所有训练与导出结果统一写入 `outputs/`：
- `outputs/water_quality_model.pth`
- `outputs/scaler_x.pkl`
- `outputs/scaler_y.pkl`
- `outputs/model_config.json`
- `outputs/training_log.txt`
- `outputs/model_data.h`

## 3. 关键脚本说明
- `new.py`：训练脚本（默认双输出 `cod + uv254`）
- `use_model.py`：命令行预测脚本（输出 `cod`、`uv254`）
- `export_to_c.py`：导出 C 参数头文件 `outputs/model_data.h`
- `ui_app.py`：图形界面，一键训练/导出/预测
- `water_quality_ai.c/.h`：STM32 侧推理引擎（已适配 `OUTPUT_SIZE`）

## 5. 各文件运行思路（建议按顺序）

### 5.1 `new.py`（训练）
运行思路：
1. 读取 `TrainConfig`（顶部宏定义作为默认参数）。
2. 读取 Excel 并解析输入/输出列（支持场景切换），输出数据统计与相关系数。
3. 划分训练/测试集，执行 `MinMaxScaler(-1,1)` 归一化。
4. 构建 BP 网络（`tansig + purelin`），可选 PSO 先优化初始权重。
5. 按 `adam`/`gd`/`trainlm_like` 训练，带验证损失监控、早停与学习率调度。
6. 用 `evaluate_regression` 计算 `MSE/RMSE/R2/MAPE`（总体 + 分输出），输出过拟合诊断。
7. 输出测试集逐样本预测值，便于人工检查。
8. 保存模型、归一化器、配置到 `outputs/`。

### 5.2 `use_model.py`（命令行推理）
运行思路：
1. 从 `outputs/` 加载 `model_config.json + pth + scaler`。
2. 若配置缺字段，使用脚本内训练参数宏定义做兜底。
3. 接收一行输入值（默认空格分隔），做归一化后前向推理。
4. 将输出反归一化后按目标字段打印结果。
5. 可输入 `q` 退出。

### 5.3 `export_to_c.py`（参数导出）
运行思路：
1. 读取 `outputs/` 中已训练好的模型与归一化器。
2. 提取 FC 层权重/偏置与归一化参数。
3. 生成 `outputs/model_data.h`：包含维度宏、权重数组、归一化数组。
4. STM32 工程只需包含该头文件即可获得同一组参数。

### 5.4 `ui_app.py`（图形工具）
运行思路：
1. 按钮 1 调 `new.py` 训练。
2. 按钮 2 调 `export_to_c.py` 导出。
3. 按钮 3 从 `outputs/` 加载模型并动态生成输入框。
4. 点击预测后执行与 `use_model.py` 等价的推理流程。
5. 日志区域显示训练/导出/预测过程输出。

### 5.5 `water_quality_ai.c/.h`（STM32 推理接口）
运行思路：
1. 在 C 侧包含 `model_data.h`（由导出脚本生成）。
2. `WaterQuality_Predict_Array`：输入归一化 -> FC+tansig -> FC -> 输出反归一化。
3. `WaterQuality_Predict3` 为 3 输入快捷封装。
4. `WaterQuality_Predict` 为旧接口兼容封装（双输出场景）。

### 5.6 `main.c`（说明）
- 当前 `main.c` 是演示/历史代码风格，与 `water_quality_ai.c` 不是同一实现链路。
- 实际 STM32 工程建议以 `water_quality_ai.c/.h + outputs/model_data.h` 为准。

## 9. STM32 调用接口说明
### 9.1 推荐接口（通用）
- `WaterQuality_Predict_Array(const float* input, float* out_values)`
  - `input` 长度为 `INPUT_SIZE`
  - `out_values` 长度为 `OUTPUT_SIZE`

### 9.2 三输入快捷接口
- `WaterQuality_Predict3(float in_0, float in_1, float in_2, float* out_values)`

### 9.3 兼容旧接口
- `WaterQuality_Predict(float in_254, float in_550, float in_tem, float* out_cod, float* out_uv254)`
- 当模型 `OUTPUT_SIZE >= 2` 时，会分别输出 `cod` 和 `uv254`。

## 10. 典型 C 侧调用片段
```c
float in[INPUT_SIZE] = {0.35f, 0.55f, 25.0f};
float out[OUTPUT_SIZE] = {0};

WaterQuality_Predict_Array(in, out);

float cod = out[0];
float uv254 = (OUTPUT_SIZE > 1) ? out[1] : 0.0f;
```

## 11. 参数建议

### 11.1 训练核心参数（`new.py` 顶部宏定义）
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `hidden_size` | 3 | 隐层神经元数，小数据集建议 2~4 |
| `learning_rate` | 0.01 | 学习率（Adam 推荐 0.001~0.01） |
| `max_epochs` | 10000 | 最大训练轮数 |
| `target_mse` | 1e-4 | 训练 MSE 早停目标 |
| `train_algo` | adam | 优化器：adam/gd/trainlm_like |
| `weight_decay` | 5e-3 | L2 正则化系数，防过拟合 |
| `early_stop_patience` | 800 | 验证损失不降多少轮后停止 |
| `lr_factor` | 0.5 | 学习率衰减因子 |
| `lr_patience` | 300 | 验证损失不降多少轮后衰减 LR |

### 11.2 PSO 参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `use_pso_init` | True | 是否启用 PSO 预搜索 |
| `pso_particles` | 30 | 粒子数量 |
| `pso_iters` | 100 | PSO 迭代次数 |
| `pso_w_max/min` | 0.9/0.4 | 惯性权重范围 |
| `pso_c1/c2` | 1.49445 | 认知/社会加速系数 |

### 11.3 数据质量与预期效果
当前数据集（47 条）的局限性：
- **254nm** 与 COD 相关系数 0.62，是唯一有效预测因子
- **550nm** 变异系数仅 0.003（几乎恒定），预测贡献极小
- **tem** 与 COD 相关系数仅 0.12
- **uv254** 与各输入均无显著相关性（|r| < 0.14），预测难度大

预期效果（hidden=3, adam, wd=5e-3）：
- COD Test R² ≈ 0.4~0.5，MAPE ≈ 15~25%
- UV254 Test R² ≈ 0.6~0.7（但 MAPE 较高，因数值本身小）
- 如需更好效果，需增加样本量或引入更多相关输入特征

- C/STM32 端闭环：`new.py` -> `export_to_c.py` -> `water_quality_ai.c/.h + model_data.h`。

## 13. 训练结果评价与优化建议

### 13.1 当前模型性能评级

| 维度 | 评级 | 说明 |
|------|:----:|------|
| 泛化能力（过拟合控制） | ⭐⭐⭐⭐ | Train-Test R² 差距仅 0.12，正则化+早停奏效 |
| COD 预测精度 | ⭐⭐ | R²=0.41, MAPE=19%，弱-中等可用 |
| UV254 预测精度 | ⭐ | R²=0.67 虚高，输入无相关性，实际不可信 |
| 训练效率 | ⭐⭐⭐⭐⭐ | 2683轮早停，节省73%时间 |

### 13.2 当前基准指标（hidden=3, adam, wd=5e-3, 47条数据）

```
总体:  Test R²=0.542  RMSE=1.662  MAPE=27.4%
COD:   Test R²=0.411  RMSE=2.346  MAPE=19.0%
UV254: Test R²=0.672  RMSE=0.152  MAPE=35.8%
```

### 13.3 已发现的预测行为问题

#### 问题一：COD 向均值回归（Regression to the Mean）

| 真实COD范围 | 模型表现 | 典型误差 |
|------------|----------|----------|
| 高值（10~12） | 系统性低估到 7~8 | 25~35% |
| 低值（3.5~6.5） | 系统性高估 | 7~25% |
| 中间值（~9） | 预测准确 | <5% |

> **原因**：模型学到了"COD 均值约7.5"这个先验，但未能充分捕捉254nm与COD的非线性关系。
> 这是小数据集上神经网络常见的"保守预测"现象。

#### 问题二：UV254 预测"虚高"的 R²

> UV254 与三个输入特征的最大相关系数仅 0.14（254nm），从信息论角度几乎不可预测。
> R²=0.67 并非真正学到了因果关系，而是模型捕捉到了输出空间分布的先验。
> **不建议依赖 UV254 的预测值做决策。**

### 13.4 数据层面优化建议

#### 短期（无需额外数据）

| 措施 | 预期效果 | 操作方式 |
|------|----------|----------|
| **单独训练 COD 模型** | COD R² 可能提升至 0.5+ | 修改 `target_cols=('cod',)` |
| **丢弃 550nm 输入** | 减少噪声，可能略提升 | 修改 `input_cols=('254','tem')` |
| **线性回归 baseline** | 确认神经网络是否必要 | `sklearn.linear_model.LinearRegression` |
| **K-fold 交叉验证** | 更准确评估泛化性能 | 当前仅单次 train/test split |

#### 中期（补充数据）

| 措施 | 优先级 | 说明 |
|------|:------:|------|
| **扩大样本量至 200+** | 🔴 最高 | 当前47条是主要瓶颈，至少需要100条以上 |
| **扩大输入特征变异范围** | 🔴 最高 | 550nm CV=0.3% 近乎恒定，需涵盖更宽测量范围 |
| **引入新输入特征** | 🟡 高 | 浊度、pH、电导率、溶解氧等与COD相关的指标 |
| **增加温度变化范围** | 🟡 高 | 当前33.75~35.61°C范围太窄，无法体现温度效应 |

#### 长期（方法改进）

| 措施 | 说明 |
|------|------|
| **分别建模 COD 和 UV254** | 两个目标与输入的关系完全不同，共享隐层不合理 |
| **尝试集成方法** | 随机森林、XGBoost 在小数据集上常优于神经网络 |
| **引入物理约束** | 若已知 Lambert-Beer 定律等先验，可加入网络结构 |
| **迁移学习** | 若有其他水质站点的数据，可预训练后微调 |

### 13.5 模型架构优化建议

| 措施 | 操作 | 适用场景 |
|------|------|----------|
| 尝试 hidden_size=2 | 参数降至 14，进一步防过拟合 | 样本<50 条 |
| 尝试 hidden_size=5~6 | 适度增加容量 | 样本≥100 条后 |
| 添加 BatchNorm | `nn.BatchNorm1d` 放在 tanh 前 | 训练更稳定 |
| 双隐层试探 | 3→4→3→2 结构 | 样本≥200 条后 |

### 13.6 训练策略优化建议

| 措施 | 操作 | 说明 |
|------|------|------|
| 增大 weight_decay | 5e-3 → 1e-2 | 当前仍有轻微过拟合迹象 |
| 减小 early_stop_patience | 800 → 500 | Test MSE 在~600轮已基本走平 |
| 增大 PSO 粒子数 | 30 → 50 | PSO 下降缓慢，更多粒子可能找到更优初值 |
| 尝试余弦退火 LR | 替换 ReduceLROnPlateau | 可能跳出局部最优 |

### 13.7 使用场景建议

| 场景 | 可行性 | 说明 |
|------|:------:|------|
| **COD 趋势判断**（高/中/低三档） | ✅ 可用 | 能区分 COD≈4 vs COD≈10 |
| **COD 精确定量**（±10%精度） | ❌ 不可用 | 19% MAPE，高值区系统性低估 |
| **UV254 任何预测** | ❌ 不建议 | 输入特征无预测力 |
| **STM32 部署原型验证** | ✅ 可用 | 推理链路完整，可做功能演示 |
| **生产环境部署** | ❌ 不建议 | 精度不足，需先解决数据问题 |

### 13.8 总结

> **当前模型在方法论层面已做到最优**——模型复杂度与数据量匹配、L2正则化生效、早停在合适时机终止、学习率调度正常运作。所有指标瓶颈均源于数据信息量不足（47条、输入变异小、UV254无关联特征），而非算法或代码问题。
>
> **优先级最高的行动**：收集更多高质量数据（≥100~200条），扩大550nm和温度的测量范围，引入与COD/UV254真正相关的辅助特征。
